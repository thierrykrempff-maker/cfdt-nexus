# Roadmap

## Phase 1 - Architecture durable

- Installer l'architecture professionnelle du dépôt.
- Documenter les décisions structurantes.
- Séparer agents, base documentaire, applications, workflows, tests et configuration.
- Formaliser les règles de sécurité et de gouvernance documentaire.

## Phase 2 - Agents spécialisés

- Décrire les responsabilités de chaque agent.
- Préparer les prompts de base.
- Définir les limites d'utilisation.
- Ajouter des tests de comportement pour les agents sensibles.

## Phase 3 - Base documentaire

- Structurer les sources publiques et internes.
- Classer convention, accords, jurisprudence et sources officielles.
- Définir les règles de confidentialité.
- Mettre en place une méthode de mise à jour et de vérification.

## Phase 4 - Automatisations

- Documenter les scénarios n8n.
- Prévoir les contrôles avant diffusion.
- Mettre en place une logique de suivi, logs et amélioration.
- Ajouter des automatisations GitHub de contrôle.

## Phase 5 - Applications

- Préparer l'assistant privé.
- Préparer le chatbot public.
- Préparer une interface d'administration.
- Tester les parcours avec des cas simples.

## Phase 6 - Site et veille

- Connecter les contenus utiles au site web.
- Préparer la veille automatique.
- Documenter les workflows de publication.
