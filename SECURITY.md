# Sécurité

CFDT Nexus peut manipuler des informations sensibles : situations individuelles, santé, discipline, paie, conflits de travail, documents internes ou éléments d'accords.

## Règles principales

- Ne jamais publier de document interne sans validation explicite.
- Ne jamais exposer de données personnelles dans une zone publique.
- Séparer les contenus publics et internes dans `knowledge-base/`.
- Limiter les automatismes sur les sujets sensibles.
- Prévoir une validation humaine avant toute diffusion.

## Signalement

Tout problème de sécurité, confidentialité ou exposition de données doit être traité en priorité et documenté dans `docs/security/`.
