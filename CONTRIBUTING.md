# Contribution

Ce dépôt évolue par petites modifications versionnées et relues.

## Règles

- Créer un commit clair par changement logique.
- Documenter les décisions importantes dans `docs/decisions/`.
- Ne pas ajouter de données personnelles réelles dans les exemples.
- Ne pas ajouter d'accord interne, de document sensible ou de donnée salarié sans validation.
- Prévoir des tests ou une checklist lorsque le changement touche agents, workflows ou site.

## Convention de commit

Utiliser des messages courts et explicites, par exemple :

- `Ajout du Juriste V1`
- `Architecture professionnelle CFDT Nexus`
- `Documentation du workflow de veille`
